## 核心功能
- **智能研究检索**：根据用户主题自动搜索 ArXiv、Semantic Scholar 等数据库，提取相关文献摘要并总结研究方向。
- **结构化初稿生成**：基于检索结果，利用 Claude 的思维链生成分章节的论文草稿，包含摘要、引言、方法、结果、讨论等标准结构。
- **多维度自动评审**：模拟同行评议，对草稿的逻辑一致性、论据充分性、文献覆盖度、创新性等方面给出具体修改建议。
- **迭代修订引擎**：将评审意见与原始草稿一并输入，引导 Claude 进行针对性修改，保留历史版本并支持差异对比。
- **终稿定稿与导出**：自动生成符合 LaTeX 或 Word 模板的最终文档，同时输出参考文献列表（BibTeX）。
- **交互式 CLI**：用户可在任一阶段介入调整方向，支持“研究-写作-评审-修订”循环的人机协作模式。

## 架构设计
采用管道-过滤器架构，将学术研究流程分解为连续的阶段，每个阶段由一个专门的技能模块处理，模块间通过共享的状态上下文（memory）传递数据。

**数据流**：
`用户输入` → `研究模块（文献检索与知识提取）` → `写作模块（生成草稿）` → `评审模块（自动评估）` → `修订模块（应用反馈）` → `终稿模块（格式化输出）`，其中修订后可能重新进入评审形成闭环。

**核心组件**：
- `SkillManager`：动态加载各阶段技能提示文件，注入 Claude 系统消息。
- `ResearchAgent`：封装检索 API 调用与结果清洗。
- `WritingAgent`：维护章节大纲并调用 Claude 生成内容。
- `ReviewAgent`：使用结构化的评审 prompt 模板，输出评分与建议。
- `RevisionAgent`：对比评审意见与原文，执行增量修改。
- `Orchestrator`：控制整体工作流状态跳转，支持断点续作。

**设计模式**：状态模式（处理流程分支）、策略模式（可替换的评审标准）、模板方法（各阶段遵循统一执行接口）。
