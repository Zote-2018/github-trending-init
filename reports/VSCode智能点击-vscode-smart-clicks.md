## 项目概述
`vscode-smart-clicks` 是一个 VS Code 扩展，用于改进编辑器中双击选择文本的行为。它解决了默认双击只能按单词边界（空格、标点）进行选择的局限性，能够根据上下文智能扩展选择范围——例如选中整个字符串、括号内内容、驼峰或蛇形命名的部分，甚至多级语法结构。开发者只需双击或多次点击，即可获得更符合预期的精确选取，大幅减少手动拖选的操作。

## 核心功能
- **智能双击选择**: 双击时根据光标所在上下文，自动扩展选择到最近的语义单元（如单词、字符串、参数、括号块等）。
- **多级扩展**: 连续双击可在不同层级间切换选择范围（例如从单词 → 字符串 → 整个语句 → 代码块）。
- **语言感知**: 针对不同编程语言（JavaScript、TypeScript、Python、HTML 等）内置解析规则，正确识别各自的语法结构。
- **自定义规则**: 允许用户通过配置自定义选择边界，例如正则表达式、分隔符列表或特定语法节点。
- **视觉反馈**: 选中范围时提供短暂的闪烁效果，让用户清晰感知被选中的区域。
- **与默认选择共存**: 不影响原本的拖选、三击选整行等操作，仅在双击事件上进行增强，可随时启用/禁用。

## 架构设计
项目采用典型的 VS Code 扩展架构，遵循命令驱动的事件模型：

1. **入口激活**: `package.json` 中注册 `activationEvents` (如 `onLanguage`), 在编辑器打开对应语言文件时激活扩展。
2. **选择拦截器**: 监听 `onDidChangeTextEditorSelection` 与鼠标双击事件 (`window.onDidChangeTextEditorSelection` 配合计时器判断双击)，当检测到双击行为时，调用核心选择引擎，计算新的选区并替换。
3. **上下文解析器**: 核心模块 `SmartSelectionProvider` 根据当前文件的语言标识，加载对应的解析器（内置的 `RegexParser`、`BracketParser`、`SemanticTokensProvider` 等）。解析器会分析光标附近文本，返回候选范围列表。
4. **状态管理**: 记录用户连续双击的次数与上一次选择范围，用于实现多级扩展。通过 `TextEditorDecorationType` 实现闪烁反馈。
5. **配置系统**: 从 VS Code 的 `workspace.getConfiguration` 读取用户自定义设置，动态调整解析器行为。
6. **测试与调试**: 使用 `vscode-test` 进行集成测试，确保扩展在不同语言、不同场景下正确工作。

数据流：用户双击 → 扩展捕获事件 → 获取光标位置与文档全文 → 根据语言调用解析器生成候选选区 → 按优先级或连续点击次数选择最佳范围 → 通过 `editor.selection` 更新选区 → 渲染装饰反馈。

## 亮点分析
- **上下文感知而不是简单正则**: 不同于仅靠单词边界的简单扩展，该项目能识别代码字符串、括号、语义符号，甚至利用 VS Code 的语言服务（如语义高亮）来获取准确语法节点，比同类工具更“懂代码”。
- **无侵入式设计**: 完美兼容 VS Code 原生双击、拖选、列选等操作，仅增强双击功能，用户学习成本极低。
- **高度可定制**: 允许用户针对特定语言、特定符号定义自己的扩展规则（如仅在 Markdown 中扩展中文句子），灵活性远超同类。
- **性能优化**: 解析过程只在必要时触发，且缓存解析结果，避免频繁读取大文件造成的卡顿。
- **开箱即用**: 内置了主流语言的规则集，安装后无需任何配置就能体验到智能选择，同时提供详细的 JSON Schema 配置提示。
- **作者口碑**: Anthony Fu (antfu) 是 Vue 核心团队成员，其项目以简洁、实用、高质著称，代码风格可作为学习 VS Code 扩展开发的最佳实践。
