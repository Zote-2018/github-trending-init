## 项目概述

Understand-Anything 是一个将任意代码或文本转化为交互式知识图谱的工具，重点强调“教育启迪”而非“表面炫酷”。它解决了开发者面对陌生大型代码库时难以快速理解结构、依赖关系和关键线索的痛点。用户不仅可以通过图谱实现可视化探索，还能借助集成的 AI 助手对图进行自然语言提问。该项目设计为与主流 AI 编程工具（如 Claude Code、Cursor、Copilot 等）无缝协作，最终目的是让复杂系统变得一目了然并易于交互。

## 核心功能

- **多源输入解析**: 支持从代码库 (TypeScript, Python, Rust 等)、Markdown 文档、API Schema (OpenAPI) 自动提取实体和关系，生成知识图谱。
- **交互式图谱可视化**: 基于力导向布局的动态节点展开/折叠，支持缩放、拖拽、聚焦，展示类、函数、文件、依赖的完整关系网。
- **语义搜索**: 对图谱内所有节点（类名、函数名、注释、文档块）进行即时模糊或语义搜索，高亮命中路径。
- **自然语言提问**: 集成 AI 助手（Knowledge Graph Agent），用户可以用自然语言询问“哪些函数调用了数据库接口”、“展示所有过时的 API”等，Agent 会将问题转化为图谱查询并返回答案。
- **无缝 AI 工具集成**: 提供与 Claude Code、Cursor、Copilot、Gemini CLI 等的专用适配器，使这些编程助手能直接引用图谱上下文，理解项目全局结构。
- **图谱导出与分享**: 生成自包含的静态度报告或可分享的交互链接，便于团队 code review 或新人 onboarding 文档。
- **实时更新**: 监控文件系统变动，当源代码修改时，图谱自动增量更新，保持与代码库同步。

## 架构设计

项目采用分层插件式架构：

1. **数据采集层 (Ingestor)**: 包含多个解析器 (TS/JS/Python 等)，利用 tree-sitter 进行 AST 分析，输出通用图模型（节点、边、属性）。
2. **图数据库层 (Graph Store)**: 内存优化的图数据结构，支持高效路径查询和图遍历（邻接表+哈希索引）。
3. **查询与Agent层**: 提供 GraphQL-like 查询接口；LLM Agent 通过 function calling 将自然语言转为结构化查询。
4. **可视化渲染层 (Renderer)**: React 组件，通过 D3 力模拟布局，使用 Web Worker 避免主线程阻塞，Canvas 渲染大量节点。
5. **集成适配层 (Adapters)**: 通过 MCP (Model Context Protocol) 或 custom protocol 与各种 IDE 和 CLI 工具通信，使外部 AI 工具能访问图谱摘要和搜索功能。

**数据流**: 源代码 → Ingestor (解析) → Graph Store → Query Engine → (1) 可视化前端直接展示, (2) 通过 Adapter 提供给外部 AI 工具作为 context。

- **“教学导向”胜过“印象导向”**: 与众不同地强调帮助人“学习”系统，而不是仅仅展示一幅炫丽但难以理解的图。节点间的关系带有语义注释，解释“为什么存在这条边”。
- **广泛集成能力**: 它不像同类工具仅提供 Web 界面，而是作为中心化“知识引擎”，通过标准化适配器与主流 AI 编程工具（Claude Code、Copilot 等）集成，极大降低优秀开发者迁移的成本。
- **语义增量更新**: 不因一个小文件修改重建整个图谱，而从 AST 级别实现细粒度差异分析，再调用增量图合并算法，适合不断迭代的大型代码库。
- **自然语言即查询**: 内置为图设计的 Agent，使得开发者无需学习新的查询语法，直接用日常语言探查复杂系统结构，降低门槛且提高 debug 效率。
- **MCP 立交桥**: 利用或主动迎合 MCP（如果仓库实现了 MCP server），让 LLM 应用（如 Claude Desktop）能直接接入项目图谱，成为 AI 的长期记忆或环境感知。
