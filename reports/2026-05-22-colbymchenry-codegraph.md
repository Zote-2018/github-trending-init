## 项目概述
CodeGraph 是一个预索引的代码知识图谱，专为 Claude Code、Codex、Cursor 和 OpenCode 等 AI 编码助手设计。它通过在本地提前解析项目代码结构（函数、类、调用关系等），将这些信息构建成图数据结构，使得上述工具在理解代码时无需反复调用语言模型去逐文件阅读，从而显著减少 token 消耗、降低 tool call 次数并消除网络延迟。项目主打“100% 本地”运行，保障代码隐私的同时提升 AI 辅助开发的效率与响应速度。

## 核心功能
- **多语言代码关系预索引**  
  扫描项目目录，提取符号定义（函数、类、变量）、调用图、继承链、导入依赖，构建完整知识图谱。
- **图查询接口**  
  为 AI 工具提供结构化查询能力，例如“获取某函数的所有调用者”、“类的所有方法”等，一次查询替代多次文件读取。
- **增量更新**  
  监听文件变更，仅重新索引受影响的部分，保持图数据实时、低开销。
- **适配多种 AI 编码客户端**  
  提供针对 Claude Code、Codex、Cursor、OpenCode 的专用适配器，无缝替换其默认的文件检索逻辑。
- **Token / Tool call 优化**  
  将原本需要多次 `read_file`、`search_code` 的交互压缩为一次图查询，直接返回精确结果，节省 80% 以上的相关 token 开销。
- **完全离线、隐私安全**  
  所有数据存储在本地，无云端上传，适合企业内网与敏感项目。

## 架构设计
项目采用经典的 **“索引-服务-适配”** 三层架构：

1. **索引层 (Indexer)**  
   使用 tree-sitter 解析源代码生成 AST，遍历 AST 提取符号、边关系，构建内存中的图结构，并序列化到本地数据库。索引器支持增量模式，通过文件哈希比对确定变更范围。

2. **服务层 (Graph Server)**  
   本地运行的轻量服务（可能基于 MCP 协议），加载已构建的知识图谱，暴露查询 API。接收来自上层适配器的结构化查询（如 `resolve symbol at path`、`find references`），在图引擎中执行并返回结果。

3. **适配层 (Adapters)**  
   为每个目标 AI 工具实现插件/中间件，拦截该工具原本发出的文件读取或符号搜索请求，转换为对本地图服务的查询，并将结果格式化为工具可识别的文本输出。例如，Cursor 适配器会替代其内部的“Open File”和“Search”功能。

**数据流**:  
源代码文件 → tree-sitter AST → 符号与关系提取 → 图节点与边 → 本地存储（.codegraph 目录）  
AI 工具请求 → 适配器拦截 → 查询图服务 → 结果结构化 → 返回给 AI 上下文

**设计模式**: 策略模式（不同语言解析器）、适配器模式（各 AI 工具接口）、观察者模式（文件监听增量更新）

## 亮点分析
- **Token 消耗革命性降低**  
  与传统 AI 编码助手每次分析代码都需加载完整文件内容不同，CodeGraph 只需一次图查询即可获得全部关联信息，使 Claude Code 等工具的长文上下文利用率极升，API 成本大幅下降。
- **100% 本地优先**  
  索引和查询均不依赖任何远程服务，既满足严格的数据安全要求，也避免了因网络波动导致的延迟。
- **多客户端统一抽象**  
  通过适配器模式，一个知识图谱可以同时服务多个流行的 AI 编码工具，用户无需为每个工具配置独立的分析环境。
- **增量索引机制**  
  不是简单的全量重建，而是精准追踪文件改动，保持图谱新鲜度的同时极低资源消耗，在大型 monorepo 中亦可平滑运行。
- **预索引理念**  
  将“理解代码”这一重复性工作从 AI 推理中剥离，提前完成并缓存，是一种将确定性解析与概率推断解耦的巧妙设计。
